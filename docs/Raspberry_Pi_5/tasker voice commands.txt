That is actually a very useful symptom. It strongly suggests the failure happens before Tasker is even involved.

What is happening:

You say:

“light on”

Google Assistant interprets that as a native smart-home command.

So Assistant thinks:

you want to control a Google Home light
or adjust phone brightness

It never forwards the phrase to Tasker at all.

That explains:

“lights haven’t been set up yet”
brightness UI opening

Those are classic Assistant intent-matching behaviors.

Important concept

Google Assistant does intent recognition first.

Your phrase:

“light on”

is extremely generic.

Assistant classifies it as:

home automation
brightness control

NOT as:

“send this phrase to Tasker”

So the issue is probably not your webhook or Tasker profile.

The fix: use a trigger phrase Assistant does NOT hijack

This is the key.

Avoid generic phrases like:

light on
light off
brightness
kitchen light
lamp on

These are reserved territory for Assistant.

Instead use something unique like:

“activate blue mode”
“computer alpha”
“project nightfall”
“run terrace”
“banana protocol”

Seriously. Strange phrases work better.
